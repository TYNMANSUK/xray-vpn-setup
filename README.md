# xray-vpn-setup

Установщик Xray + 3x-ui для Ubuntu VPS. Ставится одной командой, идёт по шагам
и **продолжает с места обрыва**, если связь оборвалась. После установки — меню
`vpn` и автонастройка, которая тестирует SNI-цели и создаёт несколько готовых
связок, чтобы в клиенте можно было выбрать рабочую.

## Установка

На чистом Ubuntu 22.04 / 24.04 от root:

```bash
curl -fsSL https://raw.githubusercontent.com/TYNMANSUK/xray-vpn-setup/main/install.sh | bash
```

Или вручную:

```bash
wget https://raw.githubusercontent.com/TYNMANSUK/xray-vpn-setup/main/install.sh
sudo bash install.sh
```

Установка выполняется по шагам:

```
[Шаг 1/6] Swap
[Шаг 2/6] BBR + сетевые оптимизации
[Шаг 3/6] Firewall (ufw)
[Шаг 4/6] fail2ban
[Шаг 5/6] 3x-ui + Xray
[Шаг 6/6] Финал (команда vpn, данные панели)
```

Каждый шаг сначала проверяет, не сделан ли он уже, и только потом настраивает.

## Обрыв связи — не проблема

Скрипт запоминает пройденные шаги в `/var/lib/xray-vpn/state`. Если SSH или
интернет оборвались:

- запусти ту же команду ещё раз — установка **продолжит с незавершённого шага**;
- уже готовые шаги повторно не выполняются;
- система не ломается от повторного запуска (всё идемпотентно).

Дополнительно установка при запуске через `curl | bash` или по SSH идёт внутри
tmux-сессии `vpn`, поэтому переживает отвал SSH сама по себе:

```bash
tmux attach -t vpn          # посмотреть/продолжить
tail -f /var/log/xray-vpn-install.log
```

## Команда vpn

После установки доступны команды `vpn` и `xray-vpn`:

```bash
vpn                  # меню управления
vpn info             # данные панели и готовые ссылки
vpn setup            # автонастройка связок
vpn status           # статус сервисов, BBR, swap, порты
vpn restart          # перезапустить сервисы
vpn logs             # логи x-ui
vpn speed            # тест скорости и пинга
vpn update           # обновить скрипт с GitHub
vpn uninstall        # удалить всё
```

Пункты меню:

```
1) Автонастройка: протестировать и создать связки
2) Показать данные панели и ссылки
3) Статус
4) Перезапустить сервисы
5) Логи
6) Тест скорости и пинга
7) Сбросить пароль панели
8) Обновить скрипт (GitHub)
9) Удалить всё
0) Выход
```

## Автонастройка связок

Пункт **1** в меню (или `vpn setup`). Что делает:

1. Тестирует SNI-цели (`www.microsoft.com`, `www.apple.com`, `www.samsung.com`,
   `dl.google.com`, `www.nvidia.com`) — проверяет доступность и время
   TLS 1.3-рукопожатия, выбирает самую быструю.
2. Создаёт и **реально проверяет** несколько связок:
   - VLESS + Reality, порт 443, отпечаток **Firefox** — основная;
   - VLESS + Reality, порт 8443, отпечаток **Chrome** — для сравнения;
   - Trojan + Reality, порт 2083, отпечаток **Safari** — альтернатива.
3. Для каждой поднимает временный Xray-клиент к `127.0.0.1` и делает запрос через
   локальный SOCKS. Если рукопожатие и проксирование работают — связка остаётся,
   если нет — удаляется автоматически.
4. Открывает нужные порты в ufw, сохраняет рабочие ссылки и печатает QR-коды.

### Что проверяется, а что нет — честно

Со стороны сервера можно проверить:

- доступность и скорость SNI-цели;
- корректность конфигурации Reality (рукопожатие + проксирование трафика).

Со стороны сервера **нельзя** проверить блокировки РКН — сервер не видит DPI
твоего провайдера. Поэтому скрипт создаёт несколько связок с разными отпечатками
и портами. **Попробуй их в клиенте по очереди — рабочей будет та, что подключается
у твоего провайдера.** По опыту, отпечаток Chrome чаще режут, поэтому начинать
стоит с Firefox-ссылки.

## Возможности подготовки системы

- Swap (размер под объём RAM)
- BBR + сетевые оптимизации (`/etc/sysctl.d/99-xray-vpn.conf`)
- ufw: открыты SSH и 443 (порт панели и порты связок добавляются автоматически)
- fail2ban для SSH (backend systemd)
- Установка официального 3x-ui + Xray (MHSanaei)

## Файлы

| Файл                              | Назначение                              |
|-----------------------------------|-----------------------------------------|
| `/root/xray-vpn-info.txt`         | Данные панели и готовые ссылки          |
| `/var/log/xray-vpn-install.log`   | Полный лог установки                    |
| `/var/lib/xray-vpn/state`         | Состояние шагов (для продолжения)       |
| `/var/lib/xray-vpn/panel.env`     | Логин/пароль/порт панели (chmod 600)    |
| `/var/lib/xray-vpn/links.txt`     | Ссылки, созданные автонастройкой        |
| `/opt/xray-vpn/install.sh`        | Копия скрипта (используется командой vpn)|

## Требования

- Ubuntu 22.04 LTS или 24.04 LTS
- Права root
- Желательно чистый сервер

## После установки

- Открой панель по адресу из `vpn info`
- При желании смени пароль (`vpn` → пункт 7)
- Запусти автонастройку (`vpn setup`) и забери ссылки

## Удаление

```bash
vpn uninstall
```

Удалит 3x-ui/Xray, swap, сетевые оптимизации, команды `vpn`/`xray-vpn` и данные
(`ufw` и `fail2ban` остаются как системные).
